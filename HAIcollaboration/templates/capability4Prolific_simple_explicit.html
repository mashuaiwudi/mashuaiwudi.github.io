    How to interpret a decision tree?

    • A decision tree contains three types of nodes. (1) Root node, (2) Split (Internal) nodes, and (3) Leaf nodes. All the nodes are linked as parent-child relationship.
    • The Root node and each Split node can represent a condition (e.g., Age > 35). If the condition is met, go to the left child-node, otherwise, go to the right child-node.
    • The Leaf node represents your prediction (e.g., Label is >50k or Label is < 50k).
    • Let's interpret a decision tree example.
    • In this tree, the Root node is "Education (years) <= 13". If this condition is satisfied, then follow the "Y" link to the Split node "Age > 35" (Otherwise go to another Split node "Work hours per week > 40"). If the "Age > 35" condition is also satisfied, then go to the Lead node "Label is >50k" otherwise "Label is <= 50k".
    • Based on this tree, if a person's education is 14 years, Age is 37, Work hours per week is 50. This person will be predicted as > 50k. Just like the below decision path.

    How to modify it?

    • Every node contains three parts, (1) feature (e.g., Age, Education, workclass, etc.), relation (e.g., >, <=, is, isnot, etc.), value (e.g., 35, Private, etc.)
    • For any node, you can change the feature, relation, value as you want.
    • Note that the value is a multiple-selection, for example, if you want to set a condition "Workclass belongs to Local-gov and Private", you can select them one-by-one. The selected item will become Blue
    • If you want to cancel a selection, you need to click the item again to make its color back to Black

    Finally, once you finish the modification, you can click the Finish and Next button to proceed.
    Note that the last Leaf Node of each path needs to be your prediction, "Label is >50k or Label is <=50k".
